							Развертывание и работа с ElasticSearch.
------------------------------------------------------------------------------------------------------------

------------------------------------------------------------------------------------------------------------
------------------------------ Локальная установка через Doker ---------------------------------------------
------------------------------------------------------------------------------------------------------------

1. При помощи PowerShell, нужно пройди в директорию "project_path_folder/ELK/", 
   Например: "D:\projects\audit-service\elk\"
2. Выполнить команду "docker-compose up -d"
3. Начнется скачивание и установка ЕЛК и Кибана.
4. Далее через UI Docker-а вы сможете запускать установленные приложения

Локальные ссылки по умолчанию: 
	ElasticSearch: http://localhost:9200
	Kibana:		   http://localhost:5601
   
------------------------------------------------------------------------------------------------------------
Примечание!
------------------------------------------------------------------------------------------------------------
Если при запуске ElasticSearch через Docker будет ошибка:
	[2] bootstrap checks failed
	[1]: max file descriptors [4096] for elasticsearch process is too low, increase to at least [65535]
	[2]: max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]
	
То нужно будет выполнить следующий скрипт в PowerShell:

	1. wsl -d docker-desktop
	2. sysctl -w vm.max_map_count=262144
------------------------------------------------------------------------------------------------------------


------------------------------------------------------------------------------------------------------------
-------------------------------------------- NEST .NET -----------------------------------------------------												
------------------------------------------------------------------------------------------------------------

Для работы .NET используем оффициальный фреймворк от ElasticSearch.

Подробнее можно узнать тут:
https://www.elastic.co/guide/en/elasticsearch/client/net-api/current/nest.html#nest

------------------------------------------------------------------------------------------------------------

Для использования интерфейса NEST, принимаем его в конструкторе класса:

	private readonly IElasticClient _elasticClient;

    public MyClass(IElasticClient client)
    {
        _elasticClient = client;
    }
